# Team Collaboration

Collaborate effectively with your team using DevFlow Pro.

## Team Management

Manage team members and their access.

**Team roles:**

- Owner: Full control including billing
- Admin: Manage projects and users
- Developer: Deploy and view projects
- Viewer: Read-only access

**Features:**

- Invite members
- Assign roles
- Remove members
- Transfer ownership
- View team activity

## Team Invitations

Invite new team members via email.

**Invitation process:**

1. Enter email address
2. Select role
3. Add optional message
4. Send invitation
5. Member accepts via email link

**Invitation settings:**

- Expiration: 7 days
- Resend invitation
- Cancel invitation
- Track invitation status

## Role Permissions

Detailed permissions per role.

**Owner permissions:**

- All admin permissions
- Manage billing
- Delete team
- Transfer ownership

**Admin permissions:**

- Add/remove users
- Create projects
- Delete projects
- Manage servers
- View logs
- Configure settings

**Developer permissions:**

- Deploy projects
- View projects
- View logs
- Manage domains
- Run commands

**Viewer permissions:**

- View projects
- View deployments
- View logs (read-only)

## Custom Roles

Create custom roles with specific permissions.

**Permission categories:**

- Projects (view, create, edit, delete, deploy)
- Servers (view, create, edit, delete, access)
- Domains (view, add, edit, delete)
- Logs (view, download, delete)
- Settings (view, edit)
- Users (view, invite, remove)
- Billing (view, manage)

**Use cases:**

- Junior developer (deploy only)
- QA tester (view and rollback)
- Client access (view only specific projects)
- Contractor (limited time access)

## Project-Specific Permissions

Assign permissions per project.

**Configuration:**

1. Select team member
2. Choose project
3. Set permissions
4. Save

**Permissions:**

- No access
- View only
- Deploy only
- Full access (except delete)
- Full control
